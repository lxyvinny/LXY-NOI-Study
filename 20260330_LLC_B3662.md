你好！这道题其实是在让我们玩一个“寻找最高点”的游戏。想象一下，我们有一张方格纸，每个格子里都堆了一定高度的积木。我们的任务就是找出那些比周围都要高的“小山尖”。

为了让你更好地理解，我们把这道题拆解成几个有趣的小步骤：

---

## 1. 什么是“山峰”？

在题目里，一个格子要想成为“山峰”，它必须满足一个条件：**它的个头（海拔）要比它“上下左右”四个邻居都高。**



* 如果它在中间，它有 4 个邻居。
* 如果它在边上，它可能有 2 到 3 个邻居。
* **注意：** 必须是**严格大于**。如果它和邻居一样高，那它就不能算作山峰。

---

## 2. “地理变迁”是在做什么？

题目中提到的“地理变迁”，其实就是**交换**。
想象你手里拿着两块积木，交换就是把 A 位置的积木放到 B 位置，把 B 位置的积木放到 A 位置。

**重要提示：** 题目说一共有 $T$ 次交换。我们不需要在每次交换后都去数山峰，只需要在**所有交换都完成后**，看最后那张地图长什么样，再去数一次山峰就可以了。

---

## 3. 数学小知识：二维坐标系

这道题用到了数学里的**坐标（Coordinates）**来确定位置。

* **$n$ 行 $m$ 列：** 就像教室里的座位排表。
* **$(x, y)$：** 这里的 $x$ 代表第几排（行），$y$ 代表第几个（列）。
    * 比如 $(2, 3)$ 就是指第 2 行、第 3 列的那个格子。

### 如何找邻居？
如果你站在 $(x, y)$ 这个格子上，你的邻居坐标分别是：
1.  **上方：** $(x-1, y)$
2.  **下方：** $(x+1, y)$
3.  **左边：** $(x, y-1)$
4.  **右边：** $(x, y+1)$

---

## 4. 解题的小锦囊（思路）

1.  **第一步：搬运数据**
    先把初始的地图记录下来（可以用一个大表格，也就是编程里的“二维数组”）。
2.  **第二步：执行交换**
    按照题目给出的 $T$ 次指令，把指定位置的两个数字换过来。换完所有次后，我们就得到了最终的地图。
3.  **第三步：全场扫描**
    从第一行第一个格子开始，一个一个检查：
    * 看它是不是比“上下左右”存在的邻居都大。
    * 如果是，就把它记在小本本上，并且计数器加 1。
4.  **第四步：报告结果**
    先告诉大家你一共找到了多少个山峰，然后再把小本本上记下的坐标全部报出来。

## 5. 举个例子

看看样例 1：
```
1 2 3
1 4 2
1 1 2
```
* 看中间的 **4**：它的邻居是上面的 2、下面的 1、左边的 1、右边的 2。4 比它们都大，所以 **(2, 2)** 是山峰！
* 看右上角的 **3**：它的邻居只有左边的 2 和下面的 2。3 比它们都大，所以 **(1, 3)** 也是山峰！

你想尝试一下，如果我把 $(2, 2)$ 位置的 **4** 和 $(3, 1)$ 位置的 **1** 交换一下，现在的地图里哪里会变成山峰吗？


你的代码在逻辑上有几个比较明显的“小坑”，导致它没能正确找到山峰。对于 8 岁的孩子来说，我们可以把这些错误想象成“走错了路”：

### 1. 坐标的“语言”不通
**错误点：** `cin >> t[j]` 和 `swap(a[t[0]][t[1]], a[t[2]][t[3]])`
* **原因：** 题目里的坐标是从 **1** 开始的（比如第一行第一列是 `1 1`），但你的数组是从 **0** 开始存储的。如果你直接用题目给的坐标，会访问到错误的位置。
* **修正：** 读入坐标后，记得都要减去 1。

### 2. “山峰”的定义理解错了
**错误点：** `a[i][j] >= m && a[i][j] >= n`
* **原因：** 这一行代码的意思是“这个格子的高度是不是比地图的总行数和总列数大”。这和题目要求的“比上下左右的邻居高”完全不是一回事。
* **修正：** 你需要写一个判断，拿 `a[i][j]` 去和它的邻居（上、下、左、右）挨个比大小。

### 3. 交换数据的“小意外”
**错误点：** `for (int j = 0; j <= 4; j++) { cin >> t[j]; }`
* **原因：** 你的 `t` 数组只开了 4 个空间（下标是 0, 1, 2, 3），但你的循环却试图读入 5 个数（`j <= 4` 包括了下标 4）。而且每次交换只需要 4 个坐标。

## 修改建议的逻辑流程

我们可以给你的代码做一个“小手术”：

### 第一步：正确交换
在读入 $T$ 次交换时，这样做：
```cpp
int x1, y1, x2, y2;
cin >> x1 >> y1 >> x2 >> y2;
// 记得减 1，因为数组是从 0 开始的
swap(a[x1-1][y1-1], a[x2-1][y2-1]);
```

### 第二步：如何判断“它是山峰”？
你要检查四个方向。为了防止程序“撞墙”（超出地图边界），可以这样写逻辑：
* 如果上方有格子，我的高度必须 **大于** 上方的。
* 如果下方有格子，我的高度必须 **大于** 下方的。
* 左边、右边同理。



### 第三步：存储结果
因为题目要求先输出总数，再输出所有坐标。你可以先把找到的山峰坐标存起来（比如用一个 `vector` 或者另一个数组），最后统一打印。

---

### 数学小贴士：边界判定
在判断邻居时，一定要先看这个邻居是否存在：
* **上：** 只有当行号 `i > 0` 时，才有 `a[i-1][j]`。
* **下：** 只有当行号 `i < n-1` 时，才有 `a[i+1][j]`。
* **左：** 只有当列号 `j > 0` 时，才有 `a[i][j-1]`。
* **右：** 只有当列号 `j < m-1` 时，才有 `a[i][j+1]`。
