这段代码实现了一个非常经典的递归应用：**利用递归栈的特性实现顺序输出**。

有趣的是，这段代码的逻辑是**先递归、后打印**。这意味着程序会一直“钻”到最深处（直到 $i=0$），然后在往回走（回溯）的过程中才开始打印数字。

以下是为你添加的详细注释：

```cpp
#include <bits/stdc++.h> // 万能头文件，包含 C++ 标准库中的所有常用函数
using namespace std;

int n; // 定义全局变量 n，用于接收用户输入的终点数字

// 递归函数 A
void A(int i) {
    // 【1. 递归边界 / 终止条件】
    // 如果当前的数字减小到了 0，说明已经触底了，不需要再继续递归
    if (i == 0) {
        return; // 直接返回，结束当前层的执行，开始“回溯”
    }

    // 【2. 递归调用】
    // 先去处理比当前数字小 1 的任务：A(i - 1)
    // 注意：此时这一层的 cout 还没有执行，会被“挂起”压入栈中
    A(i - 1);

    // 【3. 回溯执行 / 输出阶段】
    // 当 A(i-1) 彻底执行完返回后，才会执行这一行
    // 由于是最深层的函数先返回，所以这里会按照 1, 2, 3...n 的顺序打印
    cout << i << '\n'; 
}

int main() {
    // 从键盘输入一个整数给 n
    cin >> n;

    // 从 n 开始发起递归调用
    // 比如输入 3，调用顺序是 A(3) -> A(2) -> A(1) -> A(0)返回
    A(n);

    return 0; // 程序正常结束
}
```

---

### 💡 为什么这段代码会按 $1, 2, 3 \dots$ 顺序输出？

你可以把递归想象成**埋地雷**和**引爆**的过程：

1.  **埋地雷（递）：** `A(n)` 还没来得及打印，就先去叫 `A(n-1)`。大家都在排队等着，谁都没打印。
2.  **触底：** 直到 `A(0)` 触发了 `return`，它是第一个安全撤离的。
3.  **引爆（归）：** * `A(1)` 发现 `A(0)` 终于回来了，于是执行自己剩下的代码，打印了 **1**，然后结束。
    * `A(2)` 发现 `A(1)` 结束了，于是打印 **2**，然后结束。
    * ... 依此类推，直到最后 `A(n)` 打印。

这就是为什么虽然你从 $n$ 开始传参，但输出却是从小到大的原因。
