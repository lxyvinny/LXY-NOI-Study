# B2095 白细胞计数

## 题目描述

医院采样了某临床病例治疗期间的白细胞数量样本 $n$ 份，用于分析某种新抗生素对该病例的治疗效果。为了降低分析误差，要先从这 $n$ 份样本中去除一个数值最大的样本和一个数值最小的样本，然后将剩余 $n-2$ 个有效样本的平均值作为分析指标。同时，为了观察该抗生素的疗效是否稳定，还要给出该平均值的误差，即所有有效样本（即不包括已扣除的两个样本）与该平均值之差的绝对值的最大值。

现在请你编写程序，根据提供的 $n$ 个样本值，计算出该病例的平均白细胞数量和对应的误差。

## 输入格式

输入的第一行是一个正整数 $n$（$2<n \le 300$），表明共有 $n$ 个样本。

以下共有 $n$ 行，每行为一个浮点数，为对应的白细胞数量，其单位为 $10^9/L$。数与数之间以一个空格分开。

## 输出格式

输出为两个浮点数，中间以一个空格分开。分别为平均白细胞数量和对应的误差，单位也是 $10^9/L$。计算结果需保留到小数点后 $2$ 位。

```
13.0
11.0

```

### 第一步：去掉“显眼包”
想象你有 5 块不同长度的积木。为了求出一个最稳妥的平均长度，我们要先去掉两个最特殊的：
1.  找出其中**最长**的那块，扔掉。
2.  找出其中**最短**的那块，也扔掉。

剩下的积木，就是我们认为“表现最正常”的选手。

### 第二步：算平均数
把剩下的积木全部加在一起，然后除以剩下积木的数量（也就是 $n-2$ 个）。
> 比如样例里有 5 个数，去掉最大的和最小的，就只剩下 3 个数了。我们把这 3 个数加起来除以 3，得到的就是**平均值**。

### 第三步：找“最调皮”的一个（计算误差）
这一步是看剩下的这些积木里，哪一个离刚才算出来的“平均值”最远。
* 算一算每个剩下的数和平均值**差了多少**（不管谁大谁小，只看距离）。
* 在这些距离里，找出**最大的那个距离**。这就是题目要求的“误差”。

---

### 举个例子（用样例数据）：
你有 5 个白细胞数值：`12.0`, `13.0`, `11.0`, `9.0`, `10.0`。

1.  **排排队，找极端：**
    * 最大的是 `13.0`（划掉！）
    * 最小的是 `9.0`（划掉！）
    * **剩下的数：** `12.0`, `11.0`, `10.0`

2.  **算平均：**
    * 加起来：$12.0 + 11.0 + 10.0 = 33.0$
    * 除以 3：$33.0 \div 3 = 11.00$（这是第一个答案）

3.  **比距离：**
    * `12.0` 离 `11.0` 差了 $1.0$
    * `11.0` 离 `11.0` 差了 $0$
    * `10.0` 离 `11.0` 差了 $1.0$
    * **最大距离**就是 $1.00$（这是第二个答案）

### 1. 什么是“去掉极值的平均数”？

在数学里，如果我们想看一组数据最普遍的样子，有时候会去掉一个最大的和一个最小的。这叫**去除极值**。

**数学公式：**
假设我们有 $n$ 个数字，把它们加起来的总和记为 $S$，最大值记为 $max$，最小值记为 $min$。
那么，剔除掉这两个“特殊选手”后的平均数（我们记作 $Average$）就是：

$$Average = \frac{S - max - min}{n - 2}$$

> **通俗理解：** 就像裁判给跳水运动员打分，为了公平，会去掉一个最高分和一个最低分，剩下的分数加起来除以剩下的人数。

---

### 2. 什么是“误差”（距离）？

题目要求的“误差”，在数学上其实是求**绝对值的最大值**。

**绝对值 (Absolute Value)：**
在数轴上，一个数到平均数的**距离**。不管这个数比平均数大还是小，距离永远是正的。
* 如果数字是 $x$，平均数是 $A$。
* 它们之间的距离（误差）就是 $|x - A|$。

**最大误差：**
我们要把剩下的每一个数字都拿出来，算出它和平均数的距离，然后看谁跑得最远。



---

### 3. 数学计算的小挑战：保留两位小数

在计算机里，浮点数（小数）有时会很长很长。题目要求“保留两位小数”，这就像我们在超市买东西看到的价格（比如 11.00 元）。

* **四舍五入：** 如果第三位小数是 5 或更大，就给第二位加 1。
* **格式化：** 在编程时，我们需要专门告诉计算机：“嘿，我只需要小数点后两个数字！”

既然我们要动手写程序了，那我们就把刚才的“数学大探险”转化成计算机能听懂的**步骤清单**。

---

### 🛠️ 第一步：准备好“篮子”（变量初始化）
在程序开始，我们需要几个“蓝子”来装数据：
* 一个变量 `sum`：用来装所有数字的总和（初始为 0）。
* 一个变量 `max_val`：用来记最大的数（初始可以设为一个很小的数）。
* 一个变量 `min_val`：用来记最小的数（初始可以设为一个很大的数）。
* 一个数组/列表：用来存入所有的样本值，方便后面找“误差”。

### 🔍 第二步：收集与筛选（循环处理）
当电脑一个一个读入数字时，我们要同步做三件事：
1.  **全员合体：** 把读到的数字加进 `sum` 里。
2.  **更新最值：** * 如果现在的数比 `max_val` 还大，就让 `max_val` 等于它。
    * 如果现在的数比 `min_val` 还小，就让 `min_val` 等于它。
3.  **入队存好：** 把数字存进数组里，留着最后算误差用。

### ➗ 第三步：算出平均分（核心计算）
这时候我们要把“大个子”和“小矮人”刨除：
1.  **有效总和：** `valid_sum = sum - max_val - min_val`
2.  **计算平均：** `avg = valid_sum / (n - 2)`

### 📏 第四步：寻找“离群王”（误差计算）
这一步需要我们回到数组里，去观察每一个数字：
1.  **跳过极值：** 在算误差时，要记得跳过刚才被剔除掉的那**一个**最大值和**一个**最小值。
    * *注意：* 如果有多个最大值（比如有两个 13.0），题目要求只去掉一个，剩下的还是要参与计算哦！
2.  **算距离：** 计算每一个有效数字与 `avg` 的差的**绝对值**（即 $|x - avg|$）。
3.  **挑最大的：** 就像找最大值一样，找出这些距离里最大的那一个，记为 `max_error`。



---

### 📝 方法总结表

| 步骤 | 动作 | 关键点 |
| :--- | :--- | :--- |
| **1. 输入** | 读入 $n$ 和所有的样本值 | 使用循环 `for` 或 `while` |
| **2. 过滤** | 找出 `max` 和 `min` | 初始值设定要正确 |
| **3. 求均值** | `(总和 - max - min) / (n - 2)` | 注意是除以 $n-2$ |
| **4. 求误差** | 找 `abs(有效值 - 均值)` 的最大者 | 记得只去掉一个最高/最低 |
| **5. 输出** | 格式化输出两个小数 | 保留两位小数 (%.2f) |

### 💡 给小朋友的最后叮嘱：
在编程时，有一个“陷阱”：如果你发现有**两个**一样的最大值，你只能扔掉其中**一个**。
**解题小窍门：** 你可以先算出平均值。然后遍历所有的数，计算每个数和平均值的距离，最后你会得到 $n$ 个距离。由于被去掉的最大值和最小值通常离平均值最远，如果你觉得分辨“去掉哪一个”太难，可以先求出所有 $n$ 个数的误差，但只在**剩下的那 $n-2$ 个数**里找最大的距离。

### 📝 白细胞计数程序逻辑（伪代码）

```text
开始

    // 1. 准备工作
    读取 样本总数 n
    创建一个 列表 nums 用来存所有的数字
    设 总和 sum = 0
    设 最大值 max_val = -1 (设为一个极小的数)
    设 最小值 min_val = 100000 (设为一个极大的数)

    // 2. 第一次巡逻：收集数据并找到极值
    从 1 到 n 循环执行：
        读取 一个数字 x
        将 x 加入 列表 nums
        sum = sum + x
        
        如果 x > max_val，那么 max_val = x
        如果 x < min_val，那么 min_val = x
    循环结束

    // 3. 计算“公平”的平均值
    有效总和 valid_sum = sum - max_val - min_val
    平均值 avg = valid_sum / (n - 2)

    // 4. 第二次巡逻：寻找谁离平均值最远
    设 最大误差 max_error = 0
    设 已跳过最大值 flag_max = 假
    设 已跳过最小值 flag_min = 假

    从 列表 nums 中取出每一个数字 x 执行：
        // 关键：我们要确保只跳过一个最大和一个最小
        如果 (x 等于 max_val 且 flag_max 是 假)：
            flag_max = 真 (跳过这个数，不参与误差计算)
            继续下一次循环
            
        如果 (x 等于 min_val 且 flag_min 是 假)：
            flag_min = 真 (跳过这个数，不参与误差计算)
            继续下一次循环
            
        // 计算剩下的数字到平均值的距离
        当前距离 = 绝对值(x - avg)
        如果 当前距离 > max_error：
            max_error = 当前距离
    循环结束

    // 5. 成果展示
    输出 avg (保留两位小数) 和 max_error (保留两位小数)

结束
```

### 🧠 逻辑要点深度解析

#### 1. 为什么要设两个“旗帜” (`flag_max`, `flag_min`)？
这是这道题最容易出错的地方！
* **场景：** 假设样本是 `10, 10, 12, 13, 13`。
* **规则：** 题目说只去掉“一个”最大值和“一个”最小值。
* **陷阱：** 如果你直接写 `如果 x == max_val 就跳过`，电脑会把两个 `13` 都扔掉。
* **对策：** 我们用“旗帜”记录。第一次遇到 `13`，扔掉它并把旗帜举起来；第二次遇到 `13` 时，看到旗帜已经举起来了，就把它当成普通队员留下来计算。

#### 2. 绝对值的妙用
在计算距离时，我们不在乎数字是比平均值大还是小。
* $|12.0 - 11.0| = 1.0$
* $|10.0 - 11.0| = 1.0$
在数学里，这保证了我们的误差永远是一个正数。

#### 3. 关于保留两位小数
在真正的代码中（比如 C++ 或 Python），你会用到类似 `fixed` 和 `setprecision(2)` 或者 `%.2f` 的指令。这就像是给数字修剪指甲，只留下最整齐的部分。
