# B3860 [语言月赛 202309] pip install

## 题目描述

`pip install` 是 Python 安装包的重要指令。Python 的包具有依赖关系，在安装某些包之前，需要先安装这些包的依赖。

假定一共有依次编号为 $1 \sim N$ 的 $N$ 个包，第 $i$ 个包共有 $K_i$ 个依赖。现在，小 F 想要安装编号为 $1$ 的包。在完成 $1$ 号包及其全部依赖的安装，一共安装了多少个包？

## 输入格式

输入共 $N+1$ 行。

输入的第一行为一个正整数 $N$。

接下来 $N$ 行，第 $i$ 行描述了编号为 $i$ 的包的依赖：

- 第 $i$ 行共有 $K_i+1$ 个数。
- 第一个数为 $K_i$。
- 接下来 $K_i$ 个数，描述了所依赖的包的编号。

## 输出格式

输出一行一个整数，表示安装的包的数量。

## 输入输出样例 #1

### 输入 #1

```
5
1 2
1 3
1 4
0
0
```

### 输出 #1

```
4
```

## 说明/提示

对于 $100\%$ 的测试数据，$1 \le N \le 5000$，$0 \le K_i < N$，保证不会构成循环依赖。

## 1. 故事化解读题目

想象你正在参加一个乐高拼搭挑战。这里有编号为 $1, 2, 3 \dots$ 的很多个乐高套装。

你的目标是：**拼好 1 号模型。**

但是规则是这样的：
* 你想拼 **1 号**，说明书上说：“在拼 1 号之前，你必须先拼好 **2 号**。”
* 你想拼 **2 号**，说明书又说：“在拼 2 号之前，你必须先拼好 **3 号**。”
* 你想拼 **3 号**，说明书说：“这个直接拼就行，不需要别的。”

这时候，为了完成 1 号，你一共拼了几个？
答案是 3 个（1号、2号、3号）。

**这道题的任务就是：** 从 1 号开始，顺着说明书找到所有“必须先拼好”的模型，最后数一数你一共打开了多少个乐高盒子。

## 2. 这里的数学知识：图论与集合

这道题虽然是编程题，但背后藏着两个重要的数学概念：

### A. 有向图 (Directed Graph)
在数学里，我们把这种“谁依赖谁”的关系画成点和箭头。
* **点**：就是每一个包。
* **箭头**：如果 1 号依赖 2 号，我们就画一个从 1 指向 2 的箭头。
* **规则**：题目说“不会构成循环依赖”，意思是你不会遇到这种情况：1号等2号，2号等3号，3号又在等1号（这样就永远拼不完了！）。

### B. 集合的去重 (Sets and Uniqueness)
这是这道题最容易掉坑的地方！
假设：
* 1 号需要 2 号和 3 号。
* 2 号也需要 3 号。

你数数的时候，**3 号只能算一个包**，不能因为它被需要了两次，你就买两个 3 号回来。在数学上，我们要找的是所有相关包组成的“集合”里元素的个数。

## 3. 怎么解这道题？（不用代码的思路）

我们可以用**“贴贴纸法”**来解决：

1.  **准备贴纸**：想象每个包对应一个盒子，一开始都没有贴纸。
2.  **锁定目标**：我们要装 1 号，所以给 1 号盒子上贴一个“待安装”的标签。
3.  **顺藤摸瓜**：
    * 看看贴了标签的盒子里，有没有还没检查过的？
    * 如果有，打开它的说明书。比如 1 号的说明书说需要 2 号和 4 号。
    * 那就在 2 号和 4 号的盒子上也贴上标签（如果已经贴过了，就不用再贴）。
4.  **重复动作**：一直找新贴标签的盒子，直到所有贴了标签的盒子里的说明书都被你看完了。
5.  **数一数**：最后，数一数有多少个盒子被贴上了标签。那个数字就是答案！

## 4. 举个例子（样例 1）

* **1 号** 依赖 **2 号**
* **2 号** 依赖 **3 号**
* **3 号** 依赖 **4 号**
* **4 号** 和 **5 号** 谁都不依赖。

**过程：**
1.  我们要 1 号。（计数：1）
2.  1 号说要 2 号，去拿 2 号。（计数：2）
3.  2 号说要 3 号，去拿 3 号。（计数：3）
4.  3 号说要 4 号，去拿 4 号。（计数：4）
5.  4 号说我没依赖了。
6.  **结果：** 1, 2, 3, 4 号都被贴了标签，一共 **4** 个！

给出伪代码

为了让小朋友或者初学者理解这个过程，我们可以把“贴标签”的动作转化成计算机能听懂的逻辑。在计算机科学中，这种一层一层找下去的方法叫做 **递归 (Recursion)** 或者 **深度优先搜索 (DFS)**。

这里有一份简单的伪代码，它展示了如何通过“检查清单”来确保每个包只被计算一次：

---

### 1. 准备工作
首先，我们需要一个笔记本来记录哪些包已经安装过了，避免重复计数。

```text
创建一个清单 checked_list (初始全部为“未安装”)
创建一个变量 total_count = 0
```

### 2. 核心逻辑：安装包的函数
我们可以定义一个动作，叫做 `install(包的编号)`：

```text
函数 install(当前包):
    1. 如果 这个包 已经在 checked_list 里是“已安装”:
       - 直接返回 (什么都不用做，因为已经装过了)
    
    2. 如果 这个包 还没安装:
       - 第一步：去说明书里看这个包有哪些【依赖包】
       - 第二步：对于每一个【依赖包】，都执行一次 install(依赖包) 
         (这就是递归：在装自己之前，先去装好欠下的债)
       
       - 第三步：当所有的依赖包都装好了，把当前包标记为“已安装”
       - 第四步：total_count = total_count + 1
```

### 3. 开始运行
因为题目要求安装 **1号包**，所以我们只需要在程序开始时喊一声：

```text
执行 install(1)
输出 total_count
```

---

### 为什么这个逻辑是正确的？

1.  **不会漏掉**：因为 `install` 函数会去查看每一个依赖，直到遇到那些“不需要依赖”的包为止。
2.  **不会重复**：通过 `checked_list`，我们保证了即使 1 号和 2 号都依赖 3 号，3 号也只会被加进 `total_count` 一次。
3.  **像剥洋葱**：你想装最外层的（1号），程序会带你一层层剥到最里面的（没有依赖的包），然后从内向外一个个装好，最后回到 1 号。
