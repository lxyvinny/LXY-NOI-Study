# B2144 阿克曼（Ackermann）函数

## 题目描述

阿克曼（Ackermann）函数 $A(m,n)$ 中，$m, n$ 定义域是非负整数（$m \le 3$，$n \le 10$），函数值定义为：

$\mathit{A}(m,n)=n+1$；（$m=0$ 时）。

$\mathit{A}(m,n)=\mathit{A}(m-1,1)$；（$m>0$、$n=0$ 时）。

$\mathit{A}(m,n)=\mathit{A}(m-1,\mathit{A}(m,n-1))$；（$m,n>0$ 时）。

## 输入格式

$m$ 和 $n$。

## 输出格式

函数值。

## 输入输出样例 #1

### 输入 #1

```
2 3
```

### 输出 #1

```
9
```

你好！很高兴能带你一起探索这道有趣的数学题。

这道题介绍了一个在计算机世界里非常出名的函数——**阿克曼函数**。它就像一个“数字加工厂”，虽然规则只有三条，但它加工数字的速度非常快，甚至会让电脑都觉得累！

别担心，虽然它看起来有点复杂，但我们可以把它想象成一个**走迷宫游戏**。

---

## 1. 认识规则：这个“加工厂”怎么运作？

我们有两个数字：$m$（我们可以叫它“层数”）和 $n$（我们可以叫它“基础数”）。

这个工厂有三条规章制度：

* **规则 ①：如果层数 $m$ 是 0**
    * **结果：** 直接把 $n$ 加 1。比如 $A(0, 3)$，结果就是 $3 + 1 = 4$。这是最轻松的情况！
* **规则 ②：如果层数 $m$ 还在，但基础数 $n$ 变成 0 了**
    * **结果：** 把层数降一级，把基础数变成 1。比如 $A(1, 0)$，它会变成 $A(0, 1)$。
* **规则 ③：如果 $m$ 和 $n$ 都大于 0**
    * **结果：** 这是一个“套娃”。你要先算出 $A(m, n-1)$ 的结果，再把这个结果当成新的基础数，和降了一级的层数 $m-1$ 组合。

---

## 2. 什么是递归？（核心数学知识）

这道题涉及到一个非常重要的概念，叫做**递归 (Recursion)**。

> **小故事理解递归：**
> 你问我：“什么是递归？”
> 我告诉你：“递归就是当有人问你什么是递归时，你告诉他：‘递归就是当有人问你什么是递归时……’”

在数学里，递归就是**“自己定义自己”**。就像规则 ③，为了算出 $A(2, 3)$，你得先去算 $A(2, 2)$，为了算 $A(2, 2)$，你得先去算 $A(2, 1)$…… 就像剥洋葱一样，一层套一层，直到剥到最核心的**规则 ①**，我们才能得到确定的数字。

---

## 3. 我们来实战演练一下

假设我们要算 $A(1, 1)$：

1.  根据**规则 ③**：$A(1, 1) = A(0, A(1, 0))$
2.  哎呀，里面那个 $A(1, 0)$ 是多少呢？根据**规则 ②**：$A(1, 0) = A(0, 1)$
3.  根据**规则 ①**：$A(0, 1) = 1 + 1 = 2$
4.  现在把 $2$ 代回第 1 步：$A(1, 1) = A(0, 2)$
5.  最后根据**规则 ①**：$A(0, 2) = 2 + 1 = 3$

所以，$A(1, 1)$ 的答案就是 **3**！

---

## 4. 为什么阿克曼函数很特别？

对于 8 岁的你来说，最酷的一点是：**这个函数增长得超级快！**

* 如果 $m=1$，它就像是在做加法。
* 如果 $m=2$，它就像是在做乘法。
* 如果 $m=3$，它就像是在做乘方（很多个数字连乘）。
* 如果 $m=4$ 以上，这个数字大到连宇宙都装不下！
