# P1464 [PacNW 1999] Function

## 题目描述

对于一个递归函数 $w(a,b,c)$


- 如果 $a \le 0$ 或 $b \le 0$ 或 $c \le 0$ 就返回值 $1$。
- 如果 $a>20$ 或 $b>20$ 或 $c>20$ 就返回 $w(20,20,20)$
- 如果 $a<b$ 并且 $b<c$ 就返回 $w(a,b,c-1)+w(a,b-1,c-1)-w(a,b-1,c)$。
- 其它的情况就返回 $w(a-1,b,c)+w(a-1,b-1,c)+w(a-1,b,c-1)-w(a-1,b-1,c-1)$


这是个简单的递归函数，但实现起来可能会有些问题。当 $a,b,c$ 均为 $15$ 时，调用的次数将非常的多。你要想个办法才行。

注意：例如 $w(30,-1,0)$ 又满足条件 $1$ 又满足条件 $2$，请按照最上面的条件来算，答案为 $1$。

## 输入格式

会有若干行。

并以 $-1,-1,-1$ 结束。

## 输出格式

输出若干行，每一行格式：

`w(a, b, c) = ans`

注意空格。

## 输入输出样例 #1

### 输入 #1

```
1 1 1
2 2 2
-1 -1 -1
```

### 输出 #1

```
w(1, 1, 1) = 2
w(2, 2, 2) = 4
```

## 说明/提示

### 数据规模与约定

保证输入的数在 $[-9223372036854775808,9223372036854775807]$ 之间，并且是整数。

保证不包括 $-1, -1, -1$ 的输入行数 $T$ 满足 $1 \leq T \leq 10 ^ 5$。

## 1. 认识这个“神奇机器” $w(a, b, c)$

告诉孩子：这台机器有三个入口 $a, b, c$，你往里面丢三个数字，它就会按照固定的规则吐出一个结果。

但是这台机器有个脾气：**它特别喜欢“偷懒”和“甩锅”**。
* 如果你给它的数字太小（小于或等于 0），它连想都不想，直接告诉你答案是 **1**。
* 如果你给它的数字太大（大于 20），它会觉得太累了，直接把数字都改成 **20** 再去算。
* 如果是不大不小的数字，它会把任务拆开，扔给它的“分身”去算，最后把分身算出来的结果加加减减。

## 2. 为什么计算会“卡住”？（引出问题）

你可以跟孩子玩个小游戏：
> “如果你问机器 $w(15, 15, 15)$ 是多少，它会分出好几个小机器去算；这些小机器又会分出更多的小小机器……最后，屋子里会挤满成千上万个小机器，大家都在满头大汗地计算。这样不仅慢，还容易出错！”



这就是题目里说的“调用次数非常多”的意思——**重复劳动太多了**。

## 3. 终极绝招：小本本记录法（记忆化）

这是解决这道题的关键，你可以这样告诉孩子：

“我们要教这台机器一个聪明的办法：**随身带个小本本**。”

1.  **先翻书：** 每次机器要算一个组合（比如 $a=5, b=5, c=5$）之前，先翻翻本子，看看以前有没有算过。
2.  **直接报答案：** 如果本子上已经写了答案，就直接读出来，不用再分出小机器去算了。
3.  **没算过才动笔：** 如果本子上没写，才去辛苦计算。算完之后，**立刻把答案记在本子上**，下次再遇到就省事了。

由于题目规定大于 20 的数字都按 20 算，所以我们的“小本本”只需要准备一个 $25 \times 25 \times 25$ 的格子阵（三维数组）就足够装下所有的秘密答案了！

---

## 4. 特别注意：机器的“说明书”顺序

这道题有个陷阱（就是那个 $w(30, -1, 0)$ 的例子）。你要提醒孩子：
**“读说明书要从第一条开始看！”**

虽然 30 很大，满足第二条规则，但 -1 已经小于 0 了，满足第一条规则。既然第一条排在前面，机器就会先执行第一条，直接报出 **1**。

### 总结给孩子的建议：
* **第一步：** 检查数字是不是太小（$\le 0$）。
* **第二步：** 检查数字是不是太大（$> 20$）。
* **第三步：** 翻翻“小本本”，看这个组合算过没有。
* **第四步：** 没算过的话，按规则计算，并把结果存进“小本本”。

## 1. 准备“小本本”（定义数组）
我们需要一个三维数组来记录已经计算过的结果。因为题目中说 $a, b, c > 20$ 时都按照 $20$ 计算，所以我们只需要准备一个大小为 $25 \times 25 \times 25$ 的数组即可（多留一点余量防止越界）。

```cpp
// 比如在 C++ 中
long long memo[25][25][25]; 
```
**注意：** 记得初始化这个数组（比如全部设为 $0$），这样我们就知道哪些位置还没写过答案。

---

## 2. 编写“智能机器”函数
函数逻辑要严格遵守题目给出的优先级顺序：

### 第一步：处理边界（直接返回）
如果满足 $a \le 0$ 或 $b \le 0$ 或 $c \le 0$，**立刻**返回 $1$。
> **坑点提示：** 这一步必须最先判断。因为输入的数据可能非常大或非常小（甚至有负数），这一步能防止程序去访问数组的负数下标。

### 第二步：处理“超大号”数字
如果 $a > 20$ 或 $b > 20$ 或 $c > 20$，就把它们都当做 $20$。
即返回 `w(20, 20, 20)`。

### 第三步：翻看“小本本”
检查 `memo[a][b][c]` 是否已经有值（不为 $0$）。
* 如果有，直接返回这个值，不再往下计算。

### 第四步：执行复杂的计算并“存本子”
根据题目剩下的两个条件进行递归计算：
1. 如果 $a < b < c$，计算并存入 `memo[a][b][c]`。
2. 否则，按最后一条规则计算并存入 `memo[a][b][c]`。

最后返回这个刚存进去的值。

## 3. 处理输入输出
题目要求处理多组数据，直到看到 `-1, -1, -1` 为止。
* **数据类型：** 题目提到的数范围非常大（$2^{63}-1$ 级别），所以必须使用 `long long`（C++）或类似的 64 位整数类型。
* **格式控制：** 输出时要严格按照 `w(a, b, c) = ans` 的格式，注意空格。

### 2. 核心机器：w(a, b, c) 函数
我们要告诉机器，拿到三个数 $a, b, c$ 后，按照下面的顺序执行：

```text
函数 w(a, b, c):
    
    // 第一步：数字太小了吗？（最优先）
    如果 a <= 0 或者 b <= 0 或者 c <= 0:
        返回 1

    // 第二步：数字太大了吗？
    如果 a > 20 或者 b > 20 或者 c > 20:
        返回 w(20, 20, 20) 的结果

    // 第三步：翻翻小本本，看算过没有
    如果 memo[a][b][c] 不是 0:
        返回 memo[a][b][c] 的值  // 算过了，直接交卷！

    // 第四步：真的要算，按规则计算并记录
    如果 a < b 并且 b < c:
        // 计算第三条规则的结果
        结果 = w(a, b, c-1) + w(a, b-1, c-1) - w(a, b-1, c)
    否则:
        // 计算第四条规则的结果
        结果 = w(a-1, b, c) + w(a-1, b-1, c) + w(a-1, b, c-1) - w(a-1, b-1, c-1)

    // 第五步：【关键】把辛苦算出来的结果记在本本上
    memo[a][b][c] = 结果
    
    返回 结果
```

### 给你的编程小贴士：
1.  **大容量口袋：** 题目里的数字非常大，算出来的结果也会很大。在 C++ 里请务必使用 `long long` 类型，不然结果会“溢出”（就像杯子太小水流出来一样，数字会变错）。
2.  **顺序不能乱：** `if (a <= 0 ...)` 必须写在最前面。如果先判断 `memo[a][b][c]`，而此时 `a` 是负数，程序会因为找不到负数下标的格子而“崩溃”。
3.  **初始化：** 确保 `memo` 数组在所有计算开始前只清空一次，而不是每算一组数据就清空一次，这样才能利用之前算好的结果。

教练的意思是，除了用 `0` 来判断“小本本”上有没有写过答案，还可以专门请两个**“小助手”**（布尔值 `bool`）来帮机器做标记。

在编程中，`bool` 只有两个值：`true`（对/有）和 `false`（错/无）。

### 为什么需要布尔值？
虽然我们之前说 `memo[a][b][c] != 0` 就可以判断算没过，但万一某次计算的结果**刚好就是 0** 呢？这时候机器就会打架：它分不清这个 `0` 是“还没算过”还是“算完的结果就是 0”。

所以，教练建议的方法是：**再准备一本专门记录“算没算过”的清单。**

---

### 思路：双重确认法

我们可以增加一个和 `memo` 一样大的布尔数组，叫 `vis`（取自 visited，意思是“访问过”）。

#### 1. 准备两个工具
* `long long memo[25][25][25];` —— 记录**具体的答案**。
* `bool vis[25][25][25];` —— 记录**是否算过**。

#### 2. 修改“翻本本”的逻辑
不再看 `memo` 是不是 0，而是看 `vis` 里的标记：

* **算之前：** 问一下 `vis[a][b][c]` 是不是 `true`？如果是，说明这个格子已经写好了，直接拿 `memo` 里的答案。
* **算完后：** 不仅要把结果存进 `memo[a][b][c]`，还要把 `vis[a][b][c]` 调成 `true`，就像在清单上打个勾。



---

### 伪代码思路微调

你可以这样修改你的函数内部：

```cpp
// 规则 1 和 2 保持不变（处理 <=0 和 >20）

// 3. 查“清单” (使用布尔数组)
if (vis[a][b][c] == true) {
    return memo[a][b][c]; // 只要打过勾，直接给答案
}

// 4. 计算过程
long long res; // 临时存一下结果
if (a < b && b < c) {
    res = W(a, b, c - 1) + W(a, b - 1, c - 1) - W(a, b - 1, c);
} else {
    res = W(a - 1, b, c) + W(a - 1, b - 1, c) + W(a - 1, b, c - 1) - W(a - 1, b - 1, c - 1);
}

// 5. 【关键】记录并打勾
memo[a][b][c] = res;    // 把结果写进第一个本子
vis[a][b][c] = true;    // 在第二个本子（清单）上打勾

return res;
```

### 这样做的好处
这是一种更严谨的习惯。在以后遇到更复杂的题目（比如结果可能是负数或 0 的时候），这种“结果数组 + 标记数组”的组合就像双保险，绝对不会出错。
